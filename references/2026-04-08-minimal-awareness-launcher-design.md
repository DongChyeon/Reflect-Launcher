# 미니멀 자각 런처 설계 문서

> 생성일: 2026-04-08

---

## 개요

**핵심 철학**: 강요하지 않는다. 숫자와 경고 대신, 시각적 변화와 맥락 정보만으로 사용자가 스스로 자신의 폰 사용 패턴을 인식하게 만든다.

**두 가지 레이어**:
- **홈 화면** — 패러다임 전환 (폰 없이 보낸 시간을 메인으로)
- **앱 서랍** — 조용한 자각 (사용량이 시각적으로 녹아 있음)

---

## 화면 구조

### 1. 홈 화면

```
┌─────────────────────────┐
│                         │
│   2h 34m                │  ← 폰 없이 보낸 시간 (메인)
│   without phone         │     값이 클수록 텍스트 선명도 ↑
│                         │
│   today #8              │  ← 오늘 홈 화면 방문 횟수 (작게)
│                         │
│   오늘의 의도 문장         │  ← 아침 첫 진입 시 입력한 문장
│                         │
│                         │
│   Messages              │  ← 홈 앱 슬롯 (최대 4개)
│   Camera                │     사용량 많을수록 텍스트 흐려짐
│   Notes                 │
│                         │
└─────────────────────────┘
```

#### 홈 화면 구성 요소

| 요소 | 설명 | 동작 |
|---|---|---|
| 폰 없이 보낸 시간 | 마지막 화면 OFF 이후 경과 시간 | 값이 클수록 텍스트 선명 · 크게, 짧을수록 흐리게 |
| 오늘 방문 횟수 `#N` | 홈 화면 진입 횟수 (당일 자정 초기화) | 구석에 작게 고정. 별도 강조 없음 |
| 오늘의 의도 | 아침 첫 진입 시 사용자가 입력한 한 줄 문장 | 앱 슬롯 위에 작게 표시. 스킵 가능 |
| 홈 앱 슬롯 | 사용자가 직접 선택한 앱 (텍스트 레이블) | 최대 4개 고정. 추가하려면 기존 앱 제거 필요 |

#### 홈 앱 슬롯 규칙

- 최대 **4개**로 강제 제한
- 사용자가 직접 선택 (롱 프레스로 변경)
- 4개가 모두 채워진 상태에서 추가 시 교체 모드 진입
- 사용량이 많을수록 텍스트 투명도 증가 (경고 아닌 시각적 피드백)

---

### 2. 아침 첫 진입 플로우

오전 5시 ~ 11시 사이, 하루 중 **첫 번째** 홈 화면 진입 시에만 동작한다.

```
폰 켬
  └→ 빈 화면 0.5초 (의식적 진입 여백)
       └→ "오늘 가장 중요한 한 가지는?" 한 줄 입력창
            ├→ 입력 후 확인  → 홈 화면 상단에 문장 표시
            └→ 스킵          → 홈 화면 진입 (문장 없음)
```

- 강제 입력 없음. 스킵해도 패널티 없음
- 입력한 문장은 자정에 자동 초기화
- 앱 서랍을 열 때도 상단에 작게 유지

---

### 3. 앱 서랍

```
┌─────────────────────────┐
│  🔍  앱 검색             │
│                         │
│  Calendar               │  ← 오늘 미사용, 선명
│  Camera         ░░░░░   │  ← 오늘 다수 사용, 흐림
│  Instagram      ░░░░░   │  ← 오늘 다수 사용, 흐림
│  Maps                   │
│  Messages       ░░      │  ← 소량 사용, 살짝 흐림
│  Notes                  │
│  ...                    │
└─────────────────────────┘
```

#### 앱 서랍 구성 요소

| 요소 | 설명 |
|---|---|
| 전체 앱 목록 | 설치된 모든 앱, 알파벳 정렬 |
| 텍스트 흐림 | 오늘 사용 시간에 비례해 투명도 증가. 숫자 표기 없음 |
| 마이크로 컨텍스트 | 앱 탭 시 `오늘 N번째` 텍스트가 0.8초간 fade in/out |
| 검색 | 실시간 앱 이름 필터. 결과 없으면 시스템 검색 fallback |

#### 마이크로 컨텍스트 규칙

- 앱을 탭하는 순간 앱 이름 아래에 `오늘 N번째` 가 작게 등장
- 0.8초 후 자연스럽게 사라짐. 앱 실행을 막지 않음
- 폰트 크기는 앱 이름의 절반 이하. 색상은 흐린 회색
- 발동 기준: 2번째 실행부터 표시 (첫 실행은 자각 대상 아님)

## 데이터 구조

| 데이터 | 저장 방식 | 초기화 주기 |
|---|---|---|
| 폰 없이 보낸 시간 | 마지막 화면 OFF 타임스탬프 | 화면 켤 때마다 갱신 |
| 오늘 방문 횟수 | SharedPreferences (Int) | 자정 초기화 |
| 오늘의 의도 | SharedPreferences (String) | 자정 초기화 |
| 앱별 오늘 사용 시간 | UsageStatsManager | 당일 기준 |
| 앱별 오늘 실행 횟수 | SharedPreferences (Map) | 자정 초기화 |
| 홈 앱 슬롯 (4개) | SharedPreferences | 사용자 변경 시 |

---

## 필요 권한

| 권한 | 용도 |
|---|---|
| `PACKAGE_USAGE_STATS` | 앱별 사용 시간 측정 (텍스트 흐림, 마이크로 컨텍스트) |
| `BIND_ACCESSIBILITY_SERVICE` | 화면 잠금, 최근 앱 버튼 (선택적) |
| 없음 | 나머지 기능 모두 일반 런처 권한으로 동작 |

## 화면 전환 구조

```
홈 화면 (HomeFragment)
  ├── 스와이프 업      → 앱 서랍 (AppDrawerFragment)
  ├── 롱 프레스        → 설정 (SettingsFragment)
  ├── 홈 앱 롱 프레스  → 앱 선택 (AppDrawerFragment, 설정 모드)
  └── 아침 첫 진입     → 의도 입력 (IntentionFragment)
```

## 의도적으로 제외한 것

| 제외 항목 | 이유 |
|---|---|
| 스크린 타임 숫자 직접 표시 | 숫자보다 시각적 변화가 덜 강압적 |
| 사용 제한 · 차단 기능 | 강요하지 않는 철학과 충돌 |
| 위젯 · 아이콘 그리드 | 미니멀리즘 훼손 |
| 알림 배지 | 불필요한 정보 제거 |
| 앱 사용 시간 상세 통계 화면 | Digital Wellbeing에 위임 |

---

## 핵심 UX 원칙 요약

1. **경고하지 않는다** — 색상, 진동, 팝업으로 사용자를 압박하지 않는다
2. **숫자보다 느낌** — 수치 대신 시각적 변화로 패턴을 전달한다
3. **강제하지 않는다** — 모든 자각 요소는 스킵 가능하거나 배경에 존재한다
4. **제한이 곧 기능** — 홈 앱 4개 제한은 버그가 아닌 철학이다
